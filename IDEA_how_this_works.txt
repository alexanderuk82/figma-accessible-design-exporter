Claro, te explico cómo sería el flujo:

El usuario seleccionará un frame cualquiera en Figma. Luego, en nuestro plugin, se mostrará una interfaz moderna y elegante. En esta interfaz habrá un botón llamado "Select this frame". Al hacer clic en él, se abrirá un modal donde aparecerán dos opciones principales: 

1. Un menú desplegable (select) para que el usuario elija qué tipo de componente es el frame seleccionado.

2. Un grupo de botones de opción (radio buttons) con categorías como "atoms", "molecules" y "organisms".

Una vez seleccionadas estas opciones, automáticamente se generará una vista previa en tiempo real del código correspondiente a ese frame. Este código incluirá su estructura semántica y elementos como etiquetas, textos, etc., directamente interpretados desde los nodos de Figma.

Cuando el usuario confirme, el modal se cerrará y aparecerá un preview visual atractivo del componente exacto. Este preview reflejará cómo se han extraído los nodos de Figma y transformado en código semántico. 

El segundo paso permitirá personalizar la accesibilidad del componente según las necesidades del usuario. Aquí podrá configurar aspectos relacionados con WCAG, como atributos ARIA, tipos, y otros elementos para mayor personalización.

El tercer y último paso será exportar el componente o copiar el código generado. Este código estará disponible en formatos como HTML, React o Web Component, junto con su respectivo CSS. Todos los estilos (márgenes, flexbox, colores, tamaños, etc.) serán extraídos directamente de los nodos de Figma para garantizar precisión y consistencia.
